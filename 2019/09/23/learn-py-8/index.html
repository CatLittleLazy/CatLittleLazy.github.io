面向对象高级编程

  • 多重继承、定制类、元类

使用slots

  • 通过MethodType给类动态添加方法

  • 声明类的时候添加特殊的slots变量可以限制class实例的动态添加

  • 父类的slots对子类不起作用，子类若定义了slots(即便子类的限制为空)，则子类的限制为父类与子类之和

使用@property(很Nice)

  • 未使用@property的私有成员属性写法

  • @property装饰器负责把一个方法变成属性调用

  • 通过@property定义只读属性

多重继承(多考虑组合而不是复杂的继承链)

  • 主线一般都是单一继承下来，比如Cat除了继承Animal
  • 如果要加入额外的功能，为Cat再加入Runnable。这样的设计成为MixIn

定制类(类中的特殊变量slots或函数名len())

  • str()，类比java中toString()

  • repr()方法用于直接打印实例时使用

  • iter()方法为类添加迭代功能，可以理解为实现了Iter接口使对象可遍历，同时还需要实现next()

  • getitem()方法实现根据下标取出元素

  • getattr方法可以用来避免获取没有属性时的报错(后面讲的有点没懂其实)

  • call方法实现类实例的自身调用

使用枚举类

  • 创建枚举类及枚举类的遍历

  • 如果需要更精确的控制可以派生出自定义枚举类

  • 使用枚举元素

使用元类

  • 定义一个hello.py的class

  • 使用from hello import Hello导入模块时，Python解释器会一次执行该模块的所有语句，执行结果就是动态创建出一个Hello的class对象

  • type()函数可以查看类型，class的类型是type，class的定义是动态创建的，我们可以使用type()方法动态创建出Hellon类

  • 动态创建于直接手写class是没有去别的，Python解释器在遇到class定义时，仅仅是扫描一下class定义的语法，然后调用type()函数创建class

  • metaclass(元类)可以用于控制类的创建行为，先创建metaclass，就可以创建类，然后创建实例，可以理解为类是metaclass的实例

  • ORM(Object Relational Mapping)对象—关系映射，在编写该框架时，由于所有的类都只能动态定义，因为只有使用者才能根据表的结构定义出对应的类，所以需要大量使用metaclass
